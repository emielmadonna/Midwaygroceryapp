# Operations Dashboard

## Dashboard Goal

The dashboard should feel like a phone app for running Midway, not like a generic admin panel.

The first screen should answer:

- What needs attention?
- Who is arriving or leaving?
- Which RV sites are open?
- What sold today?
- Are there accounting exceptions?
- What is low?
- What changed?

## Home Screen

Recommended cards:

- Today's sales
- RV occupancy tonight
- Arrivals today
- Departures today
- Open alerts
- Low-stock count
- Fuel price status
- Store open/closed status

Recommended quick actions:

- Add booking
- Block site
- Update fuel price
- Update hours
- Add announcement
- Count inventory
- Ask AI
- Review accounting exceptions

## RV Screen

Views:

- Property map
- Calendar
- Booking list
- Site list

Actions:

- View site status.
- Add manual booking.
- Move booking to another site.
- Block maintenance dates.
- Edit site amenities.
- Check in guest.
- Check out guest.
- Cancel booking.
- Refund booking with owner permission.

## Inventory Screen

Views:

- Low stock
- Categories
- Search
- Recently sold
- Needs count

Actions:

- Adjust count.
- Scan barcode later.
- Mark item ordered.
- Draft vendor order.
- Sync Square inventory.

## Sales Screen

Views:

- Today
- Week
- Month
- Top items
- RV revenue
- Payment/refund activity

Actions:

- View Square order.
- Export later.
- Ask AI for summary.

## Accounting Screen

Goal:

- Keep bookkeeping work off the owner unless something needs attention.

Views:

- Daily summary
- Payout reconciliation
- Tax summary
- Fees/refunds
- Exceptions
- Accountant packet/export

Actions:

- Review exception.
- Approve accounting batch.
- Export CSV/PDF packet.
- Sync approved batch to QuickBooks later.
- Ask AI to explain variance.

## Website Content Screen

Editable:

- Hours
- Phone
- Address
- Announcement
- Instagram handle
- Hero message
- Photos
- What we carry
- RV site descriptions
- Fuel prices if displayed

## Staff Mode

Employee home should be simpler:

- Arrivals/departures.
- Site map.
- Shift notes.
- Low-stock tasks.
- Fuel price update if allowed.
- No owner-only settings.

## Owner Mode

Owner home should include:

- Sales.
- Payments/refunds.
- Staff permissions.
- AI approvals.
- Square sync status.
- Accounting exception queue.
- Audit log.

## Design Requirements

- Mobile-first.
- Large tap targets.
- Short labels.
- Bottom-sheet details for maps and lists.
- Confirmation dialogs for risky changes.
- Clear empty states.
- Clear offline/error states.
